Three steps

From enrolment to verification
in under a minute.

No specialised hardware required beyond a USB fingerprint reader. Our SDK handles the rest.

1

Enrol once

The member scans their fingerprint(s) at enrolment — at a pharmacy, clinic, or during onboarding. The template is encrypted and stored against their account. No raw biometric data is ever stored.

2

Scan at point of service

The pharmacy or clinic places a USB fingerprint reader at the counter. The member scans when collecting — BioVerify matches in under 2 seconds and returns a JSON result to your system via webhook or API poll.

3

Log, audit, report

Every verification event is immutably logged — who, where, when, and the result. Pharmacy managers and medical aid administrators can pull full audit reports at any time from the BioVerify admin dashboard.

JavaScript — BioVerify SDK 
import BioVerify from '@bioverify/sdk';

const bv = new BioVerify({
  apiKey: 'bvk_live_xxxxxx',
  endpoint: 'https://api.bioverify.io/v1',
  deviceId: 'term-0041'          // USB reader serial
});

// Trigger a verification against GENFIN member #GF-0041882
const result = await bv.verify({
  memberId: 'GF-0041882',
  context:  'pharmacy_dispense',
  rxId:     'RX-889244'
});

if (result.matched) {
  console.log(`✓ ${result.member.name} verified — confidence ${result.score}%`);
  dispense(result.rxId);
} else {
  console.warn('Identity not confirmed — escalate to pharmacist');
}
